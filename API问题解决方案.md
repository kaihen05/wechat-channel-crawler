# 微信公众号 API 阅读量同步问题解决方案

## 问题总结

### 1. 当前问题
- ✅ API 连接成功（access_token 可以获取）
- ❌ 调用阅读量 API 失败（错误码 48001 - api unauthorized）
- ❌ 无法获取文章阅读统计数据

### 2. 根本原因

**错误码 48001 说明：**
- **含义**：API 未授权
- **原因**：当前公众号账号类型或权限不足以调用数据统计 API

### 3. 微信官方要求

根据微信公众号官方文档：

| 账号类型 | 可调用情况 |
|:---------|:---------|
| **订阅号** | ❌ 不支持（仅已认证企业主体） |
| **服务号** | ✅ 仅**已认证**的企业主体 |
| **测试号** | ❌ 不支持 |

**关键限制：**
1. 必须是**已认证**的企业主体账号
2. 未认证的账号无法调用数据统计 API
3. 个人账号无法调用该 API

### 4. API 调用限制

| 限制项 | 说明 |
|:--------|:------|
| **数据存储起始时间** | 2025-11-01，之前日期数据无效 |
| **日期范围** | **仅支持查询1天** |
| **查询时间** | 建议在**次日8点后**查询前一天数据 |
| **API 路径** | `https://api.weixin.qq.com/datacube/getarticleread`（无 cgi-bin） |
| **日期格式** | YYYY-MM-DD（如 2025-11-15） |
| **返回数据** | 返回某天**所有文章**的阅读数据，不是特定文章 |

### 5. 修复的技术问题

#### 问题 1：API 路径错误
- ❌ 错误：`https://api.weixin.qq.com/cgi-bin/datacube/getarticleread`
- ✅ 正确：`https://api.weixin.qq.com/datacube/getarticleread`

#### 问题 2：日期格式错误
- ❌ 错误：YYYYMMDD（如 20260319）
- ✅ 正确：YYYY-MM-DD（如 2025-11-15）

#### 问题 3：API 调用方式错误
- ❌ 错误：尝试查询特定文章的阅读数据
- ✅ 正确：查询某天所有文章的阅读数据，然后匹配 msgid

#### 问题 4：缺少 time 模块导入
- 已修复：在 `articles.py` 中添加了 `import time`

### 6. 解决方案

#### 方案一：升级公众号类型（推荐）

如果需要使用 API 同步阅读量，需要：

1. **账号类型要求**：
   - 从订阅号升级为服务号
   - 完成企业认证（需要营业执照、组织机构代码等）

2. **认证费用**：
   - 每年 300 元认证费

3. **认证流程**：
   - 登录微信公众平台
   - 进入"账号详情"
   - 点击"微信认证"
   - 提交企业资质材料
   - 支付认证费用
   - 等待审核（约1-3个工作日）

#### 方案二：使用公众号后台爬虫（当前方案）

保持现有的爬虫方案，通过微信公众号后台获取阅读量：

**优点：**
- 不需要认证
- 免费
- 可以获取阅读量

**缺点：**
- 依赖 Cookie（有效期约 4 小时）
- 可能会被风控
- 需要定期更新凭证

**使用方法：**

1. 在微信公众号后台配置登录凭证
2. 点击"开始同步"按钮
3. 系统会自动更新文章阅读量

#### 方案三：使用第三方 API（不推荐）

- 不稳定
- 可能违规
- 不推荐使用
