# 公众号信息获取示例

## 📝 必填字段说明

### 1. fakeid（公众号唯一标识）

#### 获取方法1：通过搜索（推荐，最简单）

**步骤：**
1. 访问你系统的管理页面：`http://localhost:8000/manage.html`
2. 在搜索框输入公众号名称，例如："游戏技术社区"
3. 点击"搜索"按钮
4. 系统会返回搜索结果，但**不直接显示fakeid**

**这个方法的限制：**
- 搜索API不返回fakeid
- 需要配合浏览器开发者工具

#### 获取方法2：通过浏览器开发者工具（最准确）

**步骤：**
1. 打开浏览器，登录微信公众号后台：https://mp.weixin.qq.com
2. 进入"素材管理" -> "图文素材"页面
3. 按F12打开开发者工具
4. 切换到 **Network（网络）** 标签
5. 刷新页面
6. 在网络请求列表中找到 `appmsg` 或 `appmsglist` 相关的请求
7. 点击该请求，查看 **Payload（载荷）** 或 **Response（响应）**
8. 搜索 `fakeid` 关键字

**示例响应：**
```json
{
  "base_resp": {
    "ret": 0
  },
  "list": [
    {
      "app_msg_id": 1234567890,
      "title": "文章标题",
      "cover": "https://...",
      "author": "作者名",
      "digest": "摘要",
      "content": "...",
      "source_url": "...",
      "create_time": 1234567890,
      "fakeid": "ojr0j6Kx5T-3vK0v5K0J9g",  // 这就是fakeid
      "account_name": "gametech"
    }
  ]
}
```

**fakeid的特点：**
- 通常是一串加密的字符串
- 长度约20-30个字符
- 包含字母、数字、下划线、横杠
- 示例：`ojr0j6Kx5T-3vK0v5K0J9g`

### 2. nickname（公众号名称）

**获取方法：**
- 直接从公众号的名称或搜索结果中获取
- 例如："游戏技术社区"、"产品经理PM"

### 3. category（分类）

**可选值（必须是以下之一）：**
- `游戏` - 游戏开发相关公众号
- `产品` - 产品设计相关公众号
- `人力` - 人力资源管理相关公众号
- `技术` - 技术前沿动态相关公众号

**分类示例：**
```
公众号名称              分类的理由
游戏技术社区          -> 游戏 (因为主要讲游戏开发)
人人都是产品经理      -> 产品 (因为讲产品设计)
人力资源圈           -> 人力 (因为讲HR管理)
技术前沿              -> 技术 (因为讲最新技术)
```

## 🔧 实际操作示例

### 场景：你想添加"游戏技术社区"这个公众号

**步骤1：通过搜索找到公众号**
1. 访问 `http://localhost:8000/manage.html`
2. 搜索框输入："游戏技术社区"
3. 点击搜索
4. 如果找到了，说明这个公众号存在

**步骤2：获取fakeid**
1. 打开微信公众号后台
2. 搜索"游戏技术社区"公众号
3. 进入该公众号的文章列表
4. 打开F12开发者工具
5. 找到文章列表的API请求
6. 在响应中找到fakeid字段

**步骤3：填写Excel**
```
fakeid: ojr0j6Kx5T-3vK0v5K0J9g  （从步骤2获取）
nickname: 游戏技术社区
category: 游戏
account_name: gametech （如果知道就填，不知道可以为空）
```

**步骤4：导入**
1. 保存Excel文件
2. 在管理页面上传
3. 点击导入

## ⚠️ 重要提示

### 如果找不到fakeid怎么办？

**方法1：手动添加（推荐）**
1. 在管理页面搜索公众号
2. 点击"添加"按钮
3. 系统会自动获取fakeid
4. 添加后，可以查询数据库获取fakeid

**方法2：联系公众号运营者**
- 直接联系公众号的运营者
- 询问他们是否知道自己的fakeid

**方法3：使用第三方工具**
- 有些第三方工具可以帮助获取公众号的fakeid
- 但需要注意安全和隐私问题

### 简化方案

如果获取fakeid太麻烦，我建议：

**方案1：先手动添加几个测试**
1. 在管理页面搜索公众号
2. 手动添加5-10个公众号
3. 测试同步功能是否正常
4. 确认没问题后再批量导入

**方案2：只导入你确认的fakeid**
1. 只导入你已经确认fakeid的公众号
2. 其他的通过手动搜索添加
