# 公众号累计阅读量功能说明

## 功能概述

已成功为文章列表添加"该公众号的累计阅读量"显示功能，该阅读量来源于公众号文章的阅读量总和。

## 新增功能

### 1. 数据库字段

**文件**: `backend/app/models/channel.py`

新增字段：
- `total_read_num` (INTEGER) - 公众号累计阅读量（所有文章的阅读量总和）

### 2. 同步逻辑更新

**文件**: `backend/app/routers/articles.py`

同步文章时，会自动计算并更新公众号的累计阅读量：
- 同步完成后，计算该公众号所有文章的阅读量总和
- 更新 `channel.total_read_num` 字段

### 3. API返回

**文件**: `backend/app/schemas/article.py`

新增字段：
- `channel_total_read_num` - 公众号累计阅读量

### 4. 前端显示

**文件**: `backend/app/static/index.html`

在文章列表中显示：
- 绿色标签显示"公众号阅读: XXX"
- 显示该公众号所有文章的累计阅读量

## 显示效果

文章列表中每个文章下方会显示：

```
[公众号名称] [学校] [作者] [文章阅读量] [公众号阅读: 67890] [时间] [分类]
```

其中：
- **文章阅读量**：单篇文章的阅读量
- **公众号阅读: 67890**：该公众号所有文章的累计阅读量（绿色标签）

## 数据更新

### 同步时自动更新

每次同步文章后，系统会自动：
1. 计算该公众号所有文章的阅读量总和
2. 更新公众号的 `total_read_num` 字段
3. 在日志中显示累计阅读量

示例日志：
```
公众号 '测试技术公众号' 同步完成，新增 5 篇文章，累计阅读量: 67890
```

### 手动更新现有数据

已提供脚本更新现有公众号的累计阅读量：

```bash
cd backend
python update_channel_reads.py
```

执行后会显示：
```
公众号: 测试公众号, 累计阅读量: 0
公众号: 测试技术公众号, 累计阅读量: 67890
...
```

## 使用方法

### 1. 查看累计阅读量

访问首页 `http://localhost:8000/`，在文章列表中可以看到每个文章下方显示的"公众号阅读"标签。

### 2. 同步时自动更新

在管理页面同步文章后，累计阅读量会自动更新。

### 3. 手动更新

如需手动更新累计阅读量，运行：
```bash
cd backend
python update_channel_reads.py
```

## 数据来源

累计阅读量的计算方式：
```
公众号累计阅读量 = 该公众号所有文章的阅读量总和
```

例如：
- 公众号A有10篇文章，每篇阅读量分别为：100, 200, 150, ...
- 公众号A的累计阅读量 = 100 + 200 + 150 + ... = 总和

## 技术实现

### 数据库查询

使用SQLAlchemy聚合查询计算总和：
```python
total_reads = db.query(func.sum(Article.read_num)).filter(
    Article.channel_id == channel.id
).scalar() or 0
```

### 前端显示

使用绿色标签突出显示：

```html
<span class="flex items-center gap-1 bg-green-100 text-green-700 px-2 py-0.5 rounded text-xs">
    公众号阅读: ${article.channel_total_read_num}
</span>
```

## 注意事项

1. **初始值为0**：新添加的公众号累计阅读量为0，直到同步文章后才更新
2. **实时计算**：每次同步时都会重新计算累计阅读量
3. **数据来源**：阅读量来源于微信公众号后台API返回的数据
4. **可能为0**：如果微信API没有返回阅读量数据，则显示为0
